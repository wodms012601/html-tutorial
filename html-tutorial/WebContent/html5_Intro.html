HTML5의 등장과 추가된 기능

목록

1. HTML5의 등장 배경

  1. W3C(World Wide Web Consortium) 재단에서는 웹 표준을 제정(html1->html2->html3->html4->html5)
  2. 웹브라우즈 벤드사들간(넷스케이프, MS, ... )의 기술표준 전쟁 발생
  3. W3C 재단이 중간에서 제대로 표준화 하지 못하여 불만을 느낀 기업들이 무분별한 플러그인을 개발
  4. 플러그인은 웹 브라우저와 연동되는 특정 프로그램을 사용자 PC에 추가로 설치해 웹 브라우저의 기능을 확장하는 기술로 액티브엑스(ActiveX), 플래시(Flash)등이 있음
  5. 이런 플러그인들은 웹브라우즈에 종속됨. 가장 많이 사용되는 액티브엑스 플러그인은 인터넷 익스플로러를 제외한 웹 브라우저에서 작동하지 않음
  6. 2004년에 다른 웹 브라우저 제조사(애플, 모질라, 오페라 소프트웨어)는 새로운 표준 제정 그룹(WHATWG)을 자체적으로 설립(2008년 구글 참여)
  7. WHATWG는 Web Application 1.0 표준을 작성
  8. 2009년에 W3C 재단과 마이크로소프트가 함께 제정하던 XHTML 2.0 표준이 붕괴하면서 대체안으로 Web Application 1.0 표준을 사용
  9. Web Application 1.0 표준을 현재 HTML5 표준이라고 부름

2. HTML5의추가된 기능들중 일부

  1. 더 분명하고 구조화된 마크업 ex) <div>대신에 <header><footer><nav>...등을 사용
  2. 플러그인에 덜 의존하는 멀티미디어 : 비디오, 오디오, 벡터 그래픽을 표현하기 위해 플러그인에 의지할 필요가 없어짐
  3. 도메인 사이의 통신 가능
  4. 웹 소켓 사용
  5. 클라이언트 측 스토리지 제공
  6. 더 나아진 인터페이스
  7. css3 사용
